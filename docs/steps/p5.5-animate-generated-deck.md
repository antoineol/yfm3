# P5.5: Animate Generated Deck & Header-Centric Redesign (DONE)

## Goal

Show the current best deck cards in real-time during optimization, animated with auto-animate. Redesign panel headers for compact, aligned layout across all three panels (Collection, Current Deck, Suggested).

## Changes

### Live deck animation
1. **Orchestrator** (`src/engine/orchestrator.ts`): Track `globalBestDeck` alongside `globalBest`, pass it as 3rd arg to `onProgress` callback. Fixed cancel race condition — now resolves ALL workers on termination, not just those with progress.
2. **Atoms** (`src/ui/lib/atoms.ts`): Add `liveBestDeckAtom`.
3. **use-optimize** (`src/ui/features/optimize/use-optimize.ts`): Wire deck atom with 1-second throttle to avoid excessive re-renders.
4. **useLiveDeckEntries** (`src/ui/features/result/use-live-deck-entries.ts`): New hook converting live deck IDs → `CardEntry[]` via `useMemo`.
5. **OptimizationProgress** (`src/ui/features/result/OptimizationProgress.tsx`): Render `CardTable` with live deck entries below progress bar.

### Header-centric panel redesign
6. **ResultPanel** (`src/ui/features/result/ResultPanel.tsx`): Title shortened to "Suggested". Icon buttons (accept=check, reject=X, rerun=redo, cancel=stop) and ATK score moved into header. Live score shows `~` prefix during optimization.
7. **SuggestedDeckComparison** (`src/ui/features/result/SuggestedDeckComparison.tsx`): Simplified to pure display — just `CardTable` + deck intelligence sections. All action/mutation logic moved to ResultPanel.
8. **CollectionPanel** (`src/ui/features/collection/CollectionPanel.tsx`): Search autocomplete moved into header with `stretch` prop. Stats badge dropped.
9. **PanelHeader** (`src/ui/components/panel-chrome.tsx`): Added `stretch` prop — when true, children fill remaining header space (used by Collection's search).
10. **StatCard** (`src/ui/features/result/StatCard.tsx`): Deleted (no longer used).

## Key decisions

- **1s throttle on deck updates**: Score updates freely (cheap text change), deck capped at 1/sec to limit React reconciliation + auto-animate DOM analysis on ~25 rows.
- **No worker changes**: Workers already send `bestDeck` in every `PROGRESS` message — data was flowing but unused.
- **Auto-animate built-in**: `CardTable` already uses `useAutoAnimate` on `<tbody>`, so card additions/removals/reorderings animate automatically.
- **Icon buttons over text buttons**: Accept/reject/rerun/cancel use small SVG icons with `aria-label` for accessibility. Keeps header compact while maintaining large touch targets (28x28).
- **Aligned table headers**: All three panels now have the same structure — title + optional controls in header, then `CardTable` directly below. This minimizes layout shift when switching between optimization states.
