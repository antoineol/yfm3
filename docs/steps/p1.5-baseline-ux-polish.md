# P1.5: Baseline UX Polish

**Priority:** P0b — Must fix before adding new features. The current UI is a test harness, not a usable product.

**Prerequisite:** P0a (Auth) is done — the User ID input has been removed from `App.tsx`.

**Why:** The existing display was built to verify the optimizer works. It has broken inputs, no visual hierarchy, and missing interactions. Adding features on top of a broken baseline compounds the problems.

## Current Issues

### A. Config Inputs Are Broken

**Bug:** Typing in the Deck Size or Fusion Depth fields has no visible effect.

**Root cause:** The inputs are controlled components bound to async Convex query state (`value={deckSize}` from `useDeckSize()`). When you type:
- Clearing the field → `Number("")` = `NaN` → fails `v >= 5` validation → mutation silently rejected
- Typing a digit → appends to current value (e.g. "40" → "403") → fails `v <= 40` → rejected
- The input snaps back to the Convex value on re-render

**Fix:** Use local state for the input value, sync to Convex on blur/enter, and initialize from the Convex value.

### B. No Visual Hierarchy

- All 3 panels look identical (same tables, same styling)
- No panel borders, backgrounds, or visual grouping
- Optimize button is buried in the collection header
- No section separators or spacing rhythm

### C. Missing Information

- No card IDs shown (players identify cards by `#123` in-game)
- No DEF stats shown anywhere
- No card type/kind indicators
- Collection panel title says "cards" but doesn't clarify "unique cards" vs "total copies"

### D. Poor Empty/Loading States

- "No deck found." — no guidance on what to do next
- "Run the optimizer to see results." — no visual placeholder
- Loading states are plain text with no spinner

### E. Layout Issues

- 3 equal-width columns wastes space — Result panel is often empty, Collection needs more room
- Tables have no hover states or row highlighting
- No zebra striping or visual rhythm in long lists
- Headers don't convey what actions are available

## Implementation Plan

### Step 1: Fix Config Inputs

**Modify:** `src/ui/components/ConfigPanel.tsx`

Replace controlled inputs bound to Convex with local-state inputs that sync on commit:

```typescript
// Local state for editing, initialized from Convex
const [localDeckSize, setLocalDeckSize] = useState(deckSize);

// Sync from Convex when it changes (and we're not editing)
useEffect(() => { setLocalDeckSize(deckSize); }, [deckSize]);

// Commit on blur or Enter
function commitDeckSize() {
  const clamped = Math.min(Math.max(localDeckSize, HAND_SIZE), DECK_SIZE);
  setLocalDeckSize(clamped);
  setDeckSize(clamped);
}
```

- `value={localDeckSize}` — input tracks local state
- `onChange` — updates local state immediately (responsive typing)
- `onBlur` / `onKeyDown Enter` — clamps and commits to Convex
- Same pattern for Fusion Depth
