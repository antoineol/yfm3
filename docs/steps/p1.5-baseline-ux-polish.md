## Design Direction: Egyptian Mysticism × Card Game Utility

This is a tool for Yu-Gi-Oh! Forbidden Memories — a PS1 game steeped in ancient Egyptian mythology. The audience is passionate retro-gaming fans who care deeply about deck optimization. The UI should feel like an artifact from the game's world: dark, atmospheric, touched with gold — not a generic gray dashboard.

**Aesthetic:** Dark midnight base with warm gold accents, subtle atmospheric depth, and card-game-specific patterns. Think ancient stone tablet meets modern data dashboard.

**Typography:**
- **Headings:** [Cinzel](https://fonts.google.com/specimen/Cinzel) — a classicist display font with Egyptian column proportions, perfect for a game set in ancient Egypt
- **Body / UI:** [Geist Sans](https://vercel.com/font) — crisp, modern sans-serif that reads well at small sizes in tables and labels
- **Numbers / Stats:** [Geist Mono](https://vercel.com/font) — monospace variant for ATK/DEF values and scores, ensuring column alignment

**Color Palette (CSS custom properties in `index.css`):**
| Token | Value | Usage |
|-------|-------|-------|
| `--bg-deep` | `#080c14` | Page background — near-black with blue undertone |

**Key Principles:**
- Gold is the accent, not blue. Blue-600 buttons on gray-900 is the default Tailwind starter — it says nothing about this app's identity.
- Warm tones throughout (off-white text, amber highlights) instead of cold neutral grays.
- Tables should feel like card inventories, not spreadsheets.
- Stats should be hero numbers that reward the user for waiting through optimization.
- Empty states should feel like the start of an adventure, not a broken page.

## Current Issues

### A. Config Inputs Are Broken

**Bug:** Typing in the Deck Size or Fusion Depth fields has no visible effect.

**Root cause:** The inputs are controlled components bound to async Convex query state (`value={deckSize}` from `useDeckSize()`). When you type:
- Clearing the field → `Number("")` = `NaN` → fails `v >= 5` validation → mutation silently rejected
- Typing a digit → appends to current value (e.g. "40" → "403") → fails `v <= 40` → rejected
- The input snaps back to the Convex value on re-render

**Fix:** Local state with commit-on-blur/enter. Keep a local `string` state initialized from the Convex value. On blur/enter, parse, clamp to valid range, and call the Convex mutation. If the Convex value changes externally (and the input isn't focused), sync it back. No need for zod or RHF — the validation is two number range checks.

### B. No Visual Hierarchy

- All 3 panels look identical (same tables, same styling)
- No panel borders, backgrounds, or visual grouping
- Optimize button is buried in the collection header
- No section separators or spacing rhythm

### C. Missing Information

- No card IDs shown (players identify cards by `#123` in-game)
- No DEF stats shown anywhere
- No card type/kind indicators
- Collection panel title says "cards" but doesn't clarify "unique cards" vs "total copies"

### D. Poor Empty/Loading States

- "No deck found." — no guidance on what to do next
- "Run the optimizer to see results." — no visual placeholder
- Loading states are plain text with no spinner

### E. Layout Issues

- 3 equal-width columns wastes space — Result panel is often empty, Collection needs more room
- Tables have no hover states or row highlighting
- No visual rhythm in long lists
- Headers don't convey what actions are available

## Implementation Plan

### Step 1: Design Foundation — Theme & Typography

**Modify:** `index.html`
- Load Cinzel (display weights 400, 700) and Geist Sans / Geist Mono from Google Fonts or self-host

**Modify:** `src/ui/index.css`
- Define the full color palette as CSS custom properties under `@theme`
- Define font-family tokens
- Add a subtle noise/grain background texture via CSS (repeating radial-gradient or inline SVG data URI — zero network requests)
- Style the scrollbar for webkit (thin, gold-tinted thumb on dark track)
- Add a `.card-row` utility with hover transition

**Why this is Step 1:** Every subsequent step references these tokens. Getting the foundation right first prevents rework.

### Step 2: Fix Config Inputs

**Modify:** `src/ui/components/ConfigPanel.tsx`

- Replace controlled inputs with local-state pattern:
  - `useState<string>` initialized from Convex value
  - `onFocus`: mark editing
  - `onBlur` / `onKeyDown Enter`: parse → clamp → mutate Convex → unfocus
  - `useEffect`: when Convex value changes and input is not focused, sync back
- Style inputs with gold focus ring (`focus:ring-1 focus:ring-[var(--gold)]`), dark surface background
- Move config into a horizontal bar with subtle bottom border — acts as a toolbar below the header
- Add inline validation feedback: if the typed value is out of range, show the input border in `--stat-atk` red

### Step 3: Layout & Panel Architecture

**Modify:** `src/ui/App.tsx`

The three equal-width columns layout is wrong:
- Collection is the largest dataset and the primary input — it deserves ~40% width
- Deck is the secondary reference — ~30%
- Result panel either shows nothing or shows the payoff — ~30%

New layout:
```
┌─────────────── Header ───────────────────┐
├─────────────── Config Bar ───────────────┤
│                                           │
│  Collection (5fr) │ Deck (4fr) │ Result (4fr) │
│                                           │
│  ┌─────────────────────────────────────┐ │
│  │         Optimize Button (full width)│ │
│  └─────────────────────────────────────┘ │
└───────────────────────────────────────────┘
```

- Use CSS grid: `grid-template-columns: 5fr 4fr 4fr` with `gap-5`
- Each panel wrapped in a card with: `bg-[var(--bg-panel)]`, `border border-[var(--border-subtle)]`, `rounded-xl`, `p-5`
- Subtle `box-shadow: 0 0 40px rgba(0,0,0,0.3)` for depth
- The Optimize button is extracted from CollectionPanel and placed between the config bar and the grid as a full-width gold CTA:
  - `bg-[var(--gold)] text-[var(--bg-deep)] font-bold uppercase tracking-wider`
  - Hover: `bg-[var(--gold-bright)]` with subtle glow (`box-shadow: 0 0 20px var(--gold-dim)`)
  - Disabled state: dimmed, no glow
  - While optimizing: pulsing gold glow animation via CSS `@keyframes`
- Responsive: below `lg` breakpoint, stack vertically

**Modify:** All panel components
- Panel headers use Cinzel font, `text-[var(--gold)]`, `uppercase`, `tracking-wide`, `text-sm`
- Subtle separator line below headers (`border-b border-[var(--border-subtle)]`)

### Step 4: Card Display Overhaul

**Modify:** `CollectionPanel.tsx`, `DeckPanel.tsx`, `ResultPanel.tsx`

Replace bare tables with styled card inventories:

**Row structure** (still a `<table>` for alignment, but styled distinctively):
```
#042  Blue-Eyes White Dragon          3000 ATK  2500 DEF   ×3
```

- **Card ID column:** `text-[var(--text-muted)] font-mono text-xs` — shown as `#042` (zero-padded to 3 digits)
- **Card name:** `text-[var(--text-primary)]` — the dominant element in each row
- **ATK value:** `text-[var(--stat-atk)] font-mono font-bold` — colored red, always visible
- **DEF value:** `text-[var(--stat-def)] font-mono text-xs` — colored blue, secondary
- **Quantity:** only shown when > 1, displayed as `×3` with `text-[var(--gold)] font-mono`

**Row behavior:**
- Hover: `bg-[var(--bg-hover)]` with 150ms transition
- Alternating row tint: even rows get a barely-visible `bg-[var(--bg-surface)]/30`
- High-ATK cards (≥2500): row has a faint left gold border (`border-l-2 border-[var(--gold-dim)]`) — subtle power indicator
- Sticky header with `bg-[var(--bg-surface)]` and bottom border

**Collection panel header:**
- Show both counts: "Collection — 42 cards (28 unique)"

### Step 5: Result Panel — The Payoff Moment

**Modify:** `src/ui/components/ResultPanel.tsx`

The result panel is where the user sees the value of waiting 60 seconds. It should feel rewarding.

**Stats display (when results exist):**
- Replace the tiny `grid-cols-2` with hero stat cards:

- Stats arranged in a single row of 2-3 highlight cards with `bg-[var(--bg-surface)]` background and `border-[var(--border-accent)]`

**Deck list** below stats uses the same card row style from Step 4.
