# P6.6 Optimize

## Goal

- Keep the simplified `p6.5` behavior, but reduce suggestion latency if the exact scan is too slow.

## Candidate Optimizations

- Rank removable cards with a cheap heuristic, then exact-score the most promising swaps first.
- Stop exact scoring early if the remaining candidates cannot beat the current best swap.
- Reuse scorer buffers aggressively across swap evaluations.
- Reduce unnecessary reruns from React data identity churn.

## Rule

- Add an optimization only after measuring that the simpler exact version is too slow.
