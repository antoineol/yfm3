# P1.1: Authentication (Replace Manual User ID)

**Priority:** P0 ā€” Must be done before P1.5 (Baseline UX Polish) since it removes the User ID input that P1.5 would otherwise need to restyle.

**Why:** The current UI exposes a raw text input for the User ID with a hardcoded default. This is a development artifact. Users should be automatically identified, like in YFM2.

## Current State (YFM3)

- `userIdAtom` is a Jotai `atomWithStorage` persisted to `localStorage` key `yfm3_user_id`
- Default value is a hardcoded string: `"user_1750627178536_oh9nqp874"`
- `App.tsx` renders a text input where the user can manually type a user ID
- All Convex hooks read `userIdAtom` and pass it to queries/mutations
- Both YFM2 and YFM3 share the same Convex deployment (`adamant-condor-151`)

## Current State (YFM2 ā€” the pattern to replicate)

- `getUserId()` in `userUtils.ts` auto-generates an ID on first visit: `user_${Date.now()}_${randomString}`
- Stored in `localStorage` key `convex_user_id`
- No UI for the user ID ā€” fully transparent
- All hooks call `getUserId()` directly

## Target State

- User ID is auto-generated on first visit and persisted in `localStorage`
- No visible User ID input in the UI
- Use the same `localStorage` key as YFM2 (`convex_user_id`) so users who already have a YFM2 session keep their data when switching to YFM3
- All existing hooks continue to work without changes (they already read from `userIdAtom`)

## Implementation Plan

### Step 1: Auto-Generate User ID

**Modify:** `src/ui/lib/atoms.ts`

Replace the hardcoded default with auto-generation:

```typescript
function generateUserId(): string {
  const random = Math.random().toString(36).slice(2, 11);
  return `user_${Date.now()}_${random}`;
}

function getOrCreateUserId(): string {
  const KEY = 'convex_user_id'; // Same key as YFM2
  const stored = localStorage.getItem(KEY);
  if (stored) return stored;
  const id = generateUserId();
  localStorage.setItem(KEY, id);
  return id;
}

// Read-only atom ā€” user ID is set once and never changes within a session
export const userIdAtom = atom(getOrCreateUserId());
```

Key changes:
- Use `atom()` instead of `atomWithStorage()` ā€” the atom is initialized from localStorage but doesn't need Jotai's storage sync since we write directly on creation
- Use `convex_user_id` as the key (shared with YFM2) for cross-app session continuity
- Remove `setUserId` usage ā€” the atom is now read-only

### Step 2: Remove User ID Input from UI

**Modify:** `src/ui/App.tsx`

- Remove the `<label>User ID: <input .../>` block
- Remove the `useSetAtom(userIdAtom)` import
- Keep `useAtomValue(userIdAtom)` only if still needed by child components (it is ā€” CollectionPanel checks `userId` for empty state)

### Step 3: Simplify Hooks (Optional Cleanup)

The current hooks all do `userId ? { userId } : "skip"`. Since `userId` is now always a non-empty string (auto-generated), the `"skip"` branch is dead code. Clean it up:

**Modify:** `src/ui/db/use-collection.ts`, `use-deck.ts`, `use-user-preferences.ts`, `use-update-preferences.ts`

```typescript
// Before
return useQuery(api.collection.getCollection, userId ? { userId } : "skip");

// After
return useQuery(api.collection.getCollection, { userId });
```

**Modify:** `src/ui/components/CollectionPanel.tsx`, `DeckPanel.tsx`

Remove the `if (!userId)` early returns ā€” userId is always present.

### Step 4: Migration ā€” Handle Existing YFM3 Users

Users who already have a `yfm3_user_id` in localStorage but no `convex_user_id` need migration:

**Modify:** `src/ui/lib/atoms.ts` (in `getOrCreateUserId`)

```typescript
function getOrCreateUserId(): string {
  const KEY = 'convex_user_id';
  const LEGACY_KEY = 'yfm3_user_id';

  const stored = localStorage.getItem(KEY);
  if (stored) return stored;

  // Migrate from legacy YFM3 key
  const legacy = localStorage.getItem(LEGACY_KEY);
  if (legacy) {
    localStorage.setItem(KEY, legacy);
    localStorage.removeItem(LEGACY_KEY);
    return legacy;
  }

  const id = generateUserId();
  localStorage.setItem(KEY, id);
  return id;
}
```
