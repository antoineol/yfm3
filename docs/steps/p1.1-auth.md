# P1.1: Authentication (Convex Auth + Google OAuth) — DONE

**Priority:** P0 — Must be done before P1.5 (Baseline UX Polish) since it removes the User ID input that P1.5 would otherwise need to restyle.

**Why:** The current UI exposes a raw text input for the User ID with a hardcoded default. This is a development artifact with no real authentication — anyone can type any user ID and access that user's data. Replace with Convex Auth + Google OAuth for real, cross-device identity.

## Decision: Why Convex Auth

| Option | Verdict |
|--------|---------|
| **Convex Auth** | **Chosen.** Native to the stack we already use. No extra service, no extra billing. Supports anonymous + OAuth. Single package. |
| Clerk | Good DX but adds an external service dependency and billing for features we don't need. |
| WorkOS | Enterprise-focused. Overkill. |
| localStorage IDs (old plan) | Not real auth. No cross-device. Data lost on cache clear. Anyone can impersonate anyone. |

**Provider:** Google OAuth. Everyone has a Google account. One click to sign in. Cross-device by definition.

## Current State

- `userIdAtom` is a Jotai `atomWithStorage` with a hardcoded default string
- `App.tsx` renders a text input where the user can manually type a user ID
- All Convex functions accept `userId: v.string()` as an argument — the client is trusted
- ~30 backend functions across the owned-card, deck, hand, and preference modules take `userId` from args
- 4 hooks in `src/ui/db/` read `userIdAtom` and pass it to queries/mutations
- No auth tables, no `auth.ts`, no `http.ts` in the Convex backend

## Target State

- User clicks "Sign in with Google" → authenticated via Convex Auth
- Same Google account on any device = same data
- Backend functions get `userId` from `getAuthUserId(ctx)` — no client-supplied userId
- `userIdAtom` is deleted — identity comes from Convex Auth context
- App is gated behind auth: unauthenticated users see a sign-in screen
- The `userId` stored in DB rows is now a Convex `Id<"users">` (still a string, so `v.string()` schema is unchanged)

## Data Migration

Existing DB rows have `userId: "user_1750627178536_oh9nqp874"` (the hardcoded dev default). After this change, new rows will have `userId: "j572abc..."` (Convex Auth user document IDs). Old data is orphaned.

**Accepted tradeoff:** This is a dev tool with a tiny user base. Re-entering a collection is easy. No migration tool needed. If a specific user needs their data moved, a one-off Convex mutation can reassign rows from the old userId to the new one.

## Implementation Plan

### Step 1: Install & Configure Convex Auth

**Install:**

```bash
bun add @convex-dev/auth @auth/core@0.37.0
npx @convex-dev/auth  # Auto-configures files + generates JWT keys
```

The `npx` command will:
- Generate `JWT_PRIVATE_KEY` and `JWKS` env vars and set them on the Convex deployment
- Create/modify `convex/auth.ts`, `convex/auth.config.ts`, `convex/http.ts`
- Prompt for `SITE_URL` (set to `http://localhost:5173` for dev)

If the auto-setup doesn't work cleanly, do it manually:

**Create:** `convex/auth.ts`

```typescript
import Google from "@auth/core/providers/google";
import { convexAuth } from "@convex-dev/auth/server";

export const { auth, signIn, signOut, store, isAuthenticated } = convexAuth({
  providers: [Google],
});
```

**Create:** `convex/auth.config.ts`

```typescript
export default {
  providers: [
    {
      domain: process.env.CONVEX_SITE_URL,
      applicationID: "convex",
    },
  ],
};
```

**Create:** `convex/http.ts`

```typescript
import { httpRouter } from "convex/server";
import { auth } from "./auth";

const http = httpRouter();
auth.addHttpRoutes(http);
export default http;
```

**Set environment variables** (via Convex dashboard or CLI):

```bash
npx convex env set SITE_URL http://localhost:5173
npx convex env set AUTH_GOOGLE_ID <google-client-id>
npx convex env set AUTH_GOOGLE_SECRET <google-client-secret>
```

Google OAuth credentials are created at [Google Cloud Console](https://console.cloud.google.com/apis/credentials) → Create OAuth 2.0 Client ID → Web application. Set the authorized redirect URI to `https://<your-convex-deployment>.convex.site/api/auth/callback/google`.

### Step 3: Create Auth Helper

**Create:** `convex/authHelper.ts`

```typescript
import { getAuthUserId } from "@convex-dev/auth/server";
import type { MutationCtx, QueryCtx } from "./_generated/server";

export async function requireAuth(ctx: QueryCtx | MutationCtx): Promise<string> {
  const userId = await getAuthUserId(ctx);
  if (!userId) throw new Error("Not authenticated");
  return userId;
}
```

Every backend function calls `requireAuth(ctx)` instead of reading `args.userId`. The returned value is an `Id<"users">` (a string) — same type as before, just sourced from auth instead of the client.

### Step 4: Migrate Backend Functions

Mechanical change across all ~30 functions in the owned-card, deck, hand, and preference modules:

**Pattern — queries:**

```typescript
// Before
export const getDeck = query({
  args: { userId: v.string() },
  handler: async (ctx, args) => {
    const deckCards = await ctx.db
      .query("deck")
      .withIndex("by_user", (q) => q.eq("userId", args.userId))
      .collect();
    return deckCards.sort((a, b) => a.cardId - b.cardId);
  },
});

// After
export const getDeck = query({
  args: {},
  handler: async (ctx) => {
    const userId = await requireAuth(ctx);
    const deckCards = await ctx.db
      .query("deck")
      .withIndex("by_user", (q) => q.eq("userId", userId))
      .collect();
    return deckCards.sort((a, b) => a.cardId - b.cardId);
  },
});
```

**Pattern — mutations:**

```typescript
// Before
export const addToDeck = mutation({
  args: { userId: v.string(), cardId: v.number() },
  handler: async (ctx, args) => {
    const { userId, cardId } = args;
    // ...
  },
});

// After
export const addToDeck = mutation({
  args: { cardId: v.number() },
  handler: async (ctx, args) => {
    const userId = await requireAuth(ctx);
    const { cardId } = args;
    // ...
  },
});
```

**Pattern — mutations with ownership checks:**

```typescript
// Before
export const removeFromDeck = mutation({
  args: { userId: v.string(), id: v.id("deck") },
  handler: async (ctx, { userId, id }) => {
    const oldDoc = await ctx.db.get(id);
    if (!oldDoc || oldDoc.userId !== userId) throw new Error("Deck card not found");
    // ...
  },
});

// After — ownership check stays, just gets userId from auth
export const removeFromDeck = mutation({
  args: { id: v.id("deck") },
  handler: async (ctx, { id }) => {
    const userId = await requireAuth(ctx);
    const oldDoc = await ctx.db.get(id);
    if (!oldDoc || oldDoc.userId !== userId) throw new Error("Deck card not found");
    // ...
  },
});
```

### Step 5: Update Client — Provider & Auth Gate

**Modify:** `src/ui/main.tsx`

import { ConvexAuthProvider } from "@convex-dev/auth/react";

**Modify:** `src/ui/App.tsx`

```typescript
import { useAuthActions } from "@convex-dev/auth/react";
import { useConvexAuth } from "convex/react";
import { CollectionPanel } from "./components/CollectionPanel.tsx";
import { ConfigPanel } from "./components/ConfigPanel.tsx";
import { DeckPanel } from "./components/DeckPanel.tsx";
import { ResultPanel } from "./components/ResultPanel.tsx";

export default function App() {
  const { isAuthenticated, isLoading } = useConvexAuth();

  if (isLoading) {
    return (
      <div className="min-h-screen bg-gray-900 text-gray-100 flex items-center justify-center">
        Loading...
      </div>
    );
  }

  if (!isAuthenticated) {
    return <SignIn />;
  }

  return (
    <div className="min-h-screen bg-gray-900 text-gray-100 p-4">
      <Header />
      <ConfigPanel />
      <div className="flex gap-6">
        <div className="flex-1"><CollectionPanel /></div>
        <div className="flex-1"><DeckPanel /></div>
        <div className="flex-1"><ResultPanel /></div>
      </div>
    </div>
  );
}

function SignIn() {
  const { signIn } = useAuthActions();
  return (
    <div className="min-h-screen bg-gray-900 text-gray-100 flex flex-col items-center justify-center gap-4">
      <h1 className="text-2xl font-bold">YFM Deck Optimizer</h1>
      <button
        type="button"
        onClick={() => void signIn("google")}
        className="px-6 py-3 bg-white text-gray-900 rounded-lg font-medium hover:bg-gray-200 transition"
      >
        Sign in with Google
      </button>
    </div>
  );
}

function Header() {
  const { signOut } = useAuthActions();
  return (
    <div className="flex items-center justify-between mb-4">
      <h1 className="text-xl font-bold">YFM Deck Optimizer</h1>
      <button
        type="button"
        onClick={() => void signOut()}
        className="text-sm text-gray-400 hover:text-gray-200"
      >
        Sign out
      </button>
    </div>
  );
}
```

**Modify:** `src/ui/db/use-update-preferences.ts`

```typescript
import { useMutation } from "convex/react";
import { api } from "../../../convex/_generated/api";
import { DECK_SIZE, HAND_SIZE, MAX_FUSION_DEPTH } from "../../engine/types/constants.ts";

export function useUpdateDeckSize() {
  const updatePreferences = useMutation(api.userPreferences.updatePreferences);
  return function setDeckSize(v: number) {
    if (v >= HAND_SIZE && v <= DECK_SIZE) {
      updatePreferences({ deckSize: v });
    }
  };
}

export function useUpdateFusionDepth() {
  const updatePreferences = useMutation(api.userPreferences.updatePreferences);
  return function setFusionDepth(v: number) {
    if (v >= 1 && v <= MAX_FUSION_DEPTH) {
      updatePreferences({ fusionDepth: v });
    }
  };
}
```
