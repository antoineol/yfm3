# P1.9: Shared Infrastructure (Preliminary)

**Priority:** P0c — Prerequisite for P2, P3, P4, P5, P6.

**Why:** Multiple upcoming features need the fusion table exposed to the UI and a reusable card search component. Building these once now avoids throwaway code and unblocks all downstream steps cleanly.

## Deliverables

### A. FusionTable Context

The fusion table (`Int16Array[722x722]`) and `cardAtk` (`Int16Array`) are currently only built inside the optimizer pipeline (`initializeBuffersBrowser()` → `OptBuffers`). The UI has no access to them.

**New file:** `src/ui/lib/fusion-table-context.tsx`

- React context that lazily builds and caches the fusion table + cardAtk
- Reuses `loadGameDataCore()` (or a subset) — parses the same CSVs, calls `buildFusionTable()`
- Exposes: `{ fusionTable: Int16Array, cardAtk: Int16Array, maxCardId: number }`
- Lazy: only builds on first access (not at app startup), then caches
- Wrap in `App.tsx` alongside the existing `CardDbProvider`

**Key decisions:**
- This duplicates the fusion table in memory vs the optimizer's copy. Acceptable: ~500KB is trivial, and the optimizer's copy lives in Web Workers anyway (separate memory space).
- Uses the same CSV imports (`?raw`) already used by `initialize-buffers-browser.ts`.
- The `cardAtk` array uses the same card ID indexing as `CardDb`, so lookups are O(1).

### B. CardAutocomplete Component

A reusable card search input used by P2 (hand selector), P3 (suggested deck editing), P5 (collection search), and P6 (fusion lookup).

**Built on:** [Base UI Combobox](https://base-ui.com/react/components/combobox) — provides keyboard navigation, accessible listbox, dropdown open/close animations, and filtering out of the box.

**New file:** `src/ui/components/CardAutocomplete.tsx`

**Props:**
```typescript
type CardAutocompleteProps = {
  /** Cards available for selection. If omitted, uses all cards from CardDb. */
  cards?: CardSpec[];
  /** Called when user selects a card */
  onSelect: (card: CardSpec) => void;
  /** Placeholder text */
  placeholder?: string;
  /** Disable the input */
  disabled?: boolean;
};
```

**Behavior:**
- Case-insensitive substring filtering against card names
- Dropdown list of matching cards (max ~20 visible, scrollable)
- Each item shows: card name + ATK
- Keyboard navigation handled by Base UI (arrow keys, Enter to select, Escape to close)
- Clears input after selection (ready for next search)
- Debounced filtering (150ms) for smooth typing

**Styling:** Use the **frontend-design** skill. Follow existing design system from P1.5 polish (Tailwind, dark theme with gold accents). Base UI is unstyled — style the popup, options, and highlights to match the app's visual language.

### C. Tab Navigation Shell

**Built on:** [Base UI Tabs](https://base-ui.com/react/components/tabs) — provides accessible tab/panel roles, keyboard navigation (arrow keys between tabs), and activation handling. No routing library needed.

**Modify:** `src/ui/App.tsx`

**Tab structure (planned for all future steps):**
- **Deck** — current 3-column layout (Collection, Deck, Results)
- **Hand** — hand fusion calculator (P2)

Future tabs (added when their step is implemented):
- **Fusions** — fusion reference/browser (P6)

**Styling:** Use the **frontend-design** skill for the tab bar. Style with Tailwind to match the dark/gold theme — active tab indicator, hover states, smooth indicator transition between tabs.

This avoids rework: the shell is in place, P2 just adds its content.
